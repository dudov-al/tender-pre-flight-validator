# Требования тендерной документации

**Проект:** Pre-flight валидатор тендерной заявки
**Федеральные законы:** 44-ФЗ, 223-ФЗ
**Дата:** 2025-02-04

---

## Кратко

Документ содержит структурированные требования тендерной документации, нарушение которых приводит к отклонению заявки. Основано на реальных причинах отклонений из практики тендерщиков.

**Ключевые показатели:**
- Средний контракт по 44-ФЗ: ~3,35 млн ₽
- Зарплата тендерного специалиста: ~80k ₽/мес (~500 ₽/час)
- Время подготовки Формы 2: несколько дней
- Частота отклонений: 7/10 даже у опытных

## 1. Требования к первой части заявки (документы)

### 1.1. Структура пакета документов

| Требование | Описание | Тип ошибки |
|---|---|---|
| **Анонимность файлов** | В файле не должно быть информации, позволяющей идентифицировать участника (логотип, имя, телефон, email, ИНН, название компании) | Блокирующее |
| **Формат файлов** | Поддерживаемые форматы (обычно: PDF, DOC, DOCX, XLS, XLSX, ZIP, SIG, XML, HTML, JPG, PNG) | Блокирующее |
| **Размер файлов** | Ограничение на размер каждого файла (обычно до 50-100 МБ) | Блокирующее |
| **Подписи файлов** | Все документы должны быть подписаны КЭП (квалифицированной электронной подписью) | Блокирующее |
| **Имена файлов** | Требования к наименованию файлов (латиница/кириллица, длина, специальные символы) | Предупреждающее |
| **Количество файлов** | Ограничение на количество загружаемых файлов | Предупреждающее |

### 1.2. Обязательные документы (по типу закупки)

**Для строительства/дорожных работ:**
- Смета
- Лицензии/допуски (СРО)
- Паспорт изделия
- Сертификаты соответствия
- ГОСТы

**Для медицинских изделий:**
- Регистрационное удостоверение
- Декларация о соответствии
- Сертификаты качества

**Для IT/услуг:**
- Решение об участии
- Предложение по срокам
- Рекомендации/портфолио (если требуется)
- Состав команды

**Для поставок товаров:**
- Паспорт изделия
- Сертификаты соответствия
- Гарантийные обязательства
- Страна происхождения

### 1.3. Контроль лишних/недостающих документов

| Тип документа | Обязательно? | Проверка |
|---|---|---|
| Решение об участии | ✅ Да | Всегда |
| Декларация соответствия | ⚠️ По ТЗ | Проверка по шаблону |
| Сертификаты | ⚠️ По ТЗ | Проверка по кодам ОКПД2 |
| Выписка из ЕГРЮЛ | ⚠️ По ТЗ | Проверка свежести |
| Доверенность (подписант ≠ руководитель) | ✅ Да | Проверка полномочий |
| Гарантийное письмо | ⚠️ По ТЗ | Проверка суммы/формулировки |

## 2. Требования ко второй части заявки (Форма 2 — характеристики)

### 2.1. Типы характеристик

| Тип | Пример | Проверка |
|---|---|---|
| **Числовые значения** | "Давление до 1,6 МПа" | Точность, единицы измерения, диапазон |
| **Диапазоны** | "от 5 до 10 м" | Минимум/максимум |
| **Текстовые значения** | "Цвет: белый/черный" | Соответствие списку |
| **Булевы значения** | "Наличие гарантии: да/нет" | да/нет |
| **Производитель** | "Производитель: XYZ" | Соответствие списку |
| **Страна происхождения** | "Китай" | Обязательное поле |

### 2.2. Ключевые правила заполнения

| Правило | Описание | Тип ошибки |
|---|---|---|
| **"Не хуже"** | Значение должно быть ≤ или ≥ указанного в ТЗ (в зависимости от параметра) | Блокирующее |
| **"Точно"** | Значение должно быть равно указанному | Блокирующее |
| **"Или эквивалент"** | Должен быть указан производитель + страна происхождения | Предупреждающее |
| **Единицы измерения** | Должны соответствовать ТЗ (мм, см, м, кг, МПа и т.д.) | Блокирующее |
| **Точность** | Количество знаков после запятой (например, "1,6 МПа", а не "1,583 МПа") | Предупреждающее |
| **Противоречия** | Значение не должно противоречить другим полям/документам | Блокирующее |
| **Обязательные поля** | Все обязательные поля должны быть заполнены | Блокирующее |

### 2.3. Контроль качества Формы 2

| Проверка | Описание |
|---|---|
| **Полнота** | Все обязательные характеристики заполнены |
| **Точность** | Значения соответствуют требованиям ТЗ |
| **Единицы** | Единицы измерения соответствуют ТЗ |
| **Формат** | Дата, числа, текст в правильном формате |
| **Противоречия** | Нет противоречий между документами и Формой 2 |
| **Версионирование** | Контроль изменений ТЗ (если были изменения) |

## 3. Технические требования к подаче

### 3.1. Требования к электронной подписи (КЭП)

| Требование | Описание | Проверка |
|---|---|---|
| **Тип подписи** | Квалифицированная электронная подпись (КЭП) | Валидность сертификата |
| **Срок действия** | Сертификат должен быть действительным на момент подачи | Проверка даты |
| **Доверенное лицо** | Если подписант ≠ руководитель — нужна доверенность | Проверка полномочий |
| **Машиночитаемость** | Доверенность должна быть машиночитаемой (для некоторых ЭТП) | Проверка формата |
| **Криптопровайдер** | Поддерживаемые криптопровайдеры (КриптоПро, VipNet и др.) | Проверка совместимости |

### 3.2. Требования к доступам и инфраструктуре

| Требование | Описание | Проверка |
|---|---|---|
| **Аккаунт на ЭТП** | Зарегистрированный аккаунт с валидными данными | Статус аккаунта |
| **Доступ к закупке** | Участник допущен к участию | Проверка статуса |
| **Баланс** | Достаточно средств на лицевом счете (для обеспечения заявки) | Проверка суммы |
| **Сертификат** | Сертификат КЭП установлен и доступен | Проверка наличия |
| **Интернет** | Стабильное подключение к интернету | Мониторинг |

### 3.3. Дедлайны и время подачи

| Тип дедлайна | Описание | Контроль |
|---|---|---|
| **Конец срока подачи** | Точное время окончания приема заявок | Напоминания за 24/6/1 час |
| **Технический тайм-аут** | Некоторым ЭТП нужно время на обработку (обычно 5-30 мин) | Подача минимум за 30 мин |
| **Смена времени** | Перевод часов/летнее время | Учет часового пояса |
| **Продление срока** | Проверка, был ли продлен срок подачи | Мониторинг изменений |

## 4. Типовые причины отклонений (с примерами)

### 4.1. Формальные ошибки

| Причина | Пример | Автоматизация |
|---|---|---|
| **Неанонимный файл** | В документе найден логотип/название компании | ✓ Автоматическое сканирование |
| **Неверный формат** | Загружен файл .exe вместо .pdf | ✓ Проверка расширения |
| **Нет подписи** | Документ не подписан КЭП | ✓ Проверка электронной подписи |
| **Превышен размер** | Файл превышает 100 МБ | ✓ Проверка размера |
| **Мало подписей** | Подписан не весь пакет документов | ✓ Проверка всех файлов |

### 4.2. Содержательные ошибки

| Причина | Пример | Автоматизация |
|---|---|---|
| **Неверное значение** | "Давление до 1,5 МПа" вместо "до 1,6 МПа" | ✓ Проверка по ТЗ |
| **Неверная единица** | "МПа" вместо "кПа" | ✓ Конвертация единиц |
| **Нет страны происхождения** | Указано только "Производитель: XYZ" | ✓ Проверка обязательного поля |
| **Противоречие** | В документе "10 шт", в Форме 2 "11 шт" | ✓ Кросс-проверка |
| **Лишний документ** | Приложено регистрационное удостоверение (не требуется по ТЗ) | ✓ Проверка по шаблону типа закупки |

### 4.3. Технические ошибки

| Причина | Пример | Автоматизация |
|---|---|---|
| **Просроченный сертификат** | Сертификат КЭП истек | ✓ Проверка даты |
| **Нет доверенности** | Подписал сотрудник, а не директор | ✓ Проверка подписанта |
| **Плохой интернет** | Подача зависла в 23:59 | ✓ Мониторинг + автосохранение |
| **Нет средств** | Недостаточно средств на обеспечение | ✓ Проверка баланса |
| **Не тот аккаунт** | Вход с аккаунта другой организации | ✓ Проверка данных аккаунта |

## 5. Чек-лист для pre-flight проверки

### 5.1. Перед загрузкой документов

- [ ] Все документы анонимны (нет логотипов, названий, контактов)
- [ ] Все файлы в поддерживаемом формате
- [ ] Размер каждого файла ≤ лимиту
- [ ] Все документы подписаны КЭП
- [ ] Сертификаты КЭП действительны
- [ ] Имена файлов соответствуют требованиям
- [ ] Все обязательные документы для типа закупки присутствуют
- [ ] Нет лишних документов (проверка по ТЗ)

### 5.2. Перед заполнением Формы 2

- [ ] Все требования из ТЗ извлечены и структурированы
- [ ] Все обязательные характеристики определены
- [ ] Понятны все единицы измерения
- [ ] Определены все "не хуже"/"точнее" правила
- [ ] Выявлены потенциальные противоречия в ТЗ

### 5.3. После заполнения Формы 2

- [ ] Все обязательные поля заполнены
- [ ] Все числовые значения соответствуют ТЗ (диапазоны, точность)
- [ ] Единицы измерения соответствуют ТЗ
- [ ] Нет противоречий между Формой 2 и документами
- [ ] Страна происхождения указана для всех товаров
- [ ] Производитель указан (если требуется)

### 5.4. Перед подачей

- [ ] КЭП сертификат действителен
- [ ] Доверенность актуальна (если подписант ≠ руководитель)
- [ ] Достаточно средств на лицевом счете
- [ ] Аккаунт на ЭТП активен и допущен
- [ ] До дедлайна минимум 30-60 минут
- [ ] Интернет-соединение стабильно
- [ ] Подготовлен пакет доказательств (скриншоты, логи)

## 6. Приоритизация требований для MVP

### 6.1. P0 (критично для первого релиза)

1. **Проверка анонимности файлов** — частая причина отклонений
2. **Проверка формата файлов** — базовая проверка
3. **Проверка подписей** — обязательно по закону
4. **Валидность сертификата КЭП** — базовая проверка
5. **Обязательные документы по типу закупки** — шаблон для 2-3 типов

### 6.2. P1 (добавить в v1.1)

1. **Проверка значений Формы 2 по ТЗ** — извлечение требований
2. **Проверка единиц измерения** — конвертация
3. **Контроль обязательных полей** — шаблон обязательных полей
4. **Кросс-проверка документов и Формы 2** — противоречия

### 6.3. P2 (после PMF)

1. **Анализ требований из PDF/DOC** — сложная NLP задача
2. **Подсказки значений из библиотеки** — требует накопления данных
3. **Режим подачи с чек-листом** — доп. функционал
4. **Журнал инцидентов** — аналитика

## 7. Интеграции

### 7.1. Для чтения документации

- **PDF/DOC парсеры** — извлечение текста и таблиц
- **ЕИС/ЭТП API** — получение исходных данных о закупке
- **Шаблоны типов закупок** — база знаний о типичных требованиях

### 7.2. Для проверок

- **КриптоПро API / КриптоАРМ** — проверка КЭП
- **Госуслуги** — проверка реестра сертификатов
- **ЕГРЮЛ API** — проверка выписок
- **Шаблоны требований** — база правил по типу закупки

### 7.3. Для мониторинга

- **ЭТП API** — проверка статуса закупки, времени до дедлайна
- **Личный кабинет ЭТП** — проверка баланса, статуса доступа
- **Календарь** — напоминания о дедлайнах
