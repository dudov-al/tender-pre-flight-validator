# Анализ реальных тендеров

**Проект:** Pre-flight валидатор тендерной заявки
**Федеральный закон:** 44-ФЗ
**Дата:** 2025-02-04

---

## Кратко

Анализ реальных тендеров и требований к заявкам на основе открытых источников (Контур.Закупки, Bicotender, 44fzrf.ru). Цель — понять структуру тендерной документации и требования к участникам.

## 1. Структура тендерной документации (по Контур.Закупки)

### Извещение о закупке — ключевой документ

**Что в извещении:**
- Все требования заказчика
- Способ определения поставщика (электронный аукцион, конкурс, запрос котировок)
- Техническое задание (ТЗ)
- Формы заявки
- Инструкция по заполнению
- Сроки подачи

**На что обратить внимание:**
- Необходимость предоставления специальных документов
- Ограничения по нацрежиму (страна происхождения товара)
- Ограничения по составу участников (СМП, СОНО)
- Техническое задание и проект контракта

### Типы закупок и структура заявки

| Тип закупки | Особенности заявки |
|---|---|
| **Электронный аукцион** | Первая часть (документы об участнике), вторая часть (характеристики товара) |
| **Конкурс** | Части аналогичны аукциону, но могут быть отличия |
| **Запрос котировок** | Упрощенная процедура, требования указаны в извещении |

### Общие этапы подготовки заявки

**Перед составлением заявки:**
1. Оценить шансы на победу (анализ заказчика, конкурентов)
2. Проанализировать предыдущие закупки заказчика
3. Проверить надежность заказчика (индикаторы "Светофора", жалобы в ФАС)
4. Проверить, можете ли выполнить ТЗ в полном объеме и сроки
5. Прочитать проект контракта

**При подготовке заявки:**
1. Внимательно изучить документы заказчика
2. Ознакомиться с инструкцией по заполнению
3. При возникновении вопросов — запросить разъяснения (не позднее чем за 3 дня до окончания срока)

**Перед подачей:**
1. Убедиться, что условия заказчика не изменились
2. Заложить время на подачу заявки
3. При ошибке — отозвать заявку и подать новую

## 2. Требования к заявке (по 44-ФЗ)

### Обязательные данные и документы (ст. 43 44-ФЗ)

Независимо от способа определения поставщика от вас потребуют:

1. **Информацию и документы об участнике закупки**
2. **Предложение по объекту закупки** (товар, работа, услуга)
3. **Информацию о стране происхождения ТРУ**

### Информация и документы об участнике

#### Группа 1: Оператор ЭТП направляет автоматически

**Данные из ЕИС (указываются при регистрации):**

| Для юрлица | Для ИП | Для физлица |
|---|---|---|
| Полное и сокращенное наименование | ФИО | ФИО |
| ИНН | ИНН | ИНН (если есть) |
| Адрес регистрации | Место жительства | Место жительства |
| Email | Email | Email |
| Телефон | Телефон | Телефон |
| Выписка из ЕГРЮЛ | Выписка из ЕГРИП | Копия документа, удостоверяющего личность |
| ФИО, ИНН, должность руководителя | — | — |
| Сведения о СМП/СОНКО | Сведения о СМП/СОНКО | — |
| Декларация о принадлежности к учреждению УИС | Декларация о принадлежности к учреждению УИС | — |
| Декларация о принадлежности к организации инвалидов | Декларация о принадлежности к организации инвалидов | — |

**Доптребования (ч. 2 и 2.1 ст. 31 44-ФЗ):**
- Опыт работы
- Наличие специалистов необходимой квалификации
- Финансовые и материальные ресурсы для исполнения контракта

**Важно:** Документы о соответствии доптребованиям нужно предоставить не менее чем за 5 дней до подачи заявки.

#### Группа 2: Указываются самостоятельно при подаче заявки

Обычно требуют:
- **Решение об одобрении крупной сделки** (если требуется)
- **Лицензии, свидетельство СРО** (при необходимости)
- **Декларацию о соответствии единым требованиям** (пп. 3-5, 7-11 ч. 1 ст. 31 44-ФЗ)
  - Не банкрот
  - Деятельность не приостановлена
  - Нет судимостей по экономическим статьям
  - Нет недоимок по налогам > 25% балансовой стоимости активов
- **Реквизиты счета**

### Предложение по объекту закупки

**Общие требования:**
1. **Страна происхождения товара** — обязательно указать, иначе заявку отклонят
2. **Не использовать расплывчатые формулировки:**
   - ❌ "от и до", "более и менее", "допускается", "должно быть"
   - ✅ Исключение: когда предусмотрены инструкцией или ГОСТом
3. **Характеристики не должны противоречить друг другу**
4. **Использовать общепринятые наименования и обозначения**
5. **Избегать формулировок с двояким смыслом**

**При закупках с проектной документацией:** достаточно согласиться на поставку ТРУ на условиях извещения.

## 3. Реестр документов по 44-ФЗ (по Bicotender)

### Основной пакет документов

Для участия в тендере необходимо подготовить:

| Документ | Примечание |
|---|---|
| Заявка на участие | Оформляется по типовому образцу |
| Выписка из ЕГРЮЛ | Не старше 6 месяцев на момент подачи |
| Документы, подтверждающие полномочия | Устав, протокол заседания о выборе директора, доверенность |
| Заверенные копии учредительных документов | — |
| Декларация за прошедший год | Финансовая отчетность |
| Копии контрактов | Свидетельствующие о соответствию требованиям |
| Сертификат банковской гарантии | Если требуется обеспечение |
| Коммерческое предложение | Предложение по объекту закупки |
| Технические аспекты (для стройки/производства) | Мини-проект, эскиз, технические характеристики |

### Общие правила оформления

**Бумажная заявка:**
- Должна быть прошита и зарегистрирована
- Запечатана в конверт
- Прилагается опись приложенных документов

**Электронная заявка:**
- Требуется ЭП (электронная подпись)
- Форма может отличаться на разных ЭТП

## 4. Единые требования к участникам (44fzrf.ru)

### Список требований (10 пунктов)

| № | Требование | Тип доказательства |
|---|---|---|
| 1 | Соответствие законодательству РФ | Декларация |
| 2 | Не ведется процедура ликвидации, нет решения о банкротстве | Декларация |
| 3 | Деятельность не приостановлена по КоАП | Декларация |
| 4 | Нет недоимок по налогам > 25% балансовой стоимости активов | Справка из налоговой |
| 5 | Нет конфликта интересов с заказчиком | Декларация |
| 6 | Нет судимостей по экономическим статьям (у руководителей) | Справка о судимости |
| 7 | Не привлекался к админответственности по ст. 19.28 КоАП (крупные сделки) | Декларация |
| 8 | Не имеет ограничений прав на результаты интеллектуальной деятельности | Декларация |
| 9 | Не является офшорной компанией | Декларация |
| 10 | Нет ограничений для участия в закупках по законодательству РФ | Декларация |

### Конфликт интересов

**Когда считается конфликтом:**
- Руководитель заказчика/член комиссии/руководитель контрактной службы/контрактный управляющий:
  - Состоит в браке с выгодоприобретателем участника
  - Является родственником (родители, дети, дедушка, бабушка, внуки, братья, сестры)
  - Владеет > 10% голосующих акций или долей в капитале участника

**Выгодоприобретатель:**
- Физическое лицо, владеющее напрямую или косвенно > 10% голосующих акций
- Или долей > 10% в уставном капитале хозяйственного общества

## 5. Типовые ошибки при подаче заявки

На основе анализа выявлены типовые причины отклонений:

### Формальные ошибки
- Не указана страна происхождения товара
- Использование расплывчатых формулировок ("от и до", "более и менее")
- Противоречия между характеристиками товара
- Несоответствие единиц измерения ТЗ
- Отсутствие обязательных документов

### Содержательные ошибки
- Не соответствие доптребованиям (опыт, специалисты, ресурсы)
- Отсутствие деклараций о соответствии единым требованиям
- Неверное заполнение данных участника
- Неверное указание полномочий подписанта

### Технические ошибки
- Просроченная ЭП
- Забыли подписать часть документов
- Превышен размер файла
- Неверный формат файла
- Подали после дедлайна

## 6. Выводы для MVP

### Что может автоматически проверять pre-flight валидатор

#### P0 (критично)
- Проверка анонимности документов (нет логотипов, названий компании)
- Проверка форматов файлов (PDF, DOC, DOCX, XLS, XLSX и др.)
- Проверка размеров файлов (≤ лимита)
- Проверка наличия ЭП на всех документах
- Проверка валидности сертификата ЭП
- Проверка свежести выписки из ЕГРЮЛ (не старше 6 месяцев)
- Проверка обязательных документов по типу закупки
- Проверка указания страны происхождения товара

#### P1 (важно)
- Проверка на расплывчатые формулировки ("от и до", "более и менее")
- Проверка на противоречия между характеристиками
- Проверка соответствия единиц измерения ТЗ
- Проверка полноты деклараций о соответствии единым требованиям
- Проверка полномочий подписанта (директор или по доверенности)
- Проверка свежести доверенности

#### P2 (полезно)
- Парсинг ТЗ из PDF/DOC для извлечения требований
- Автоматическая проверка соответствия характеристик ТЗ
- Подсказки значений из библиотеки характеристик

### Интеграции для проверок

| Проверка | Интеграция |
|---|---|
| Валидность сертификата ЭП | КриптоПро API |
| Свежесть выписки из ЕГРЮЛ | ЕГРЮЛ API |
| Проверка судимостей | Судебный реестр |
| Проверка налогов | ФНС API |
| Проверка статуса участника | ЕИС API |
