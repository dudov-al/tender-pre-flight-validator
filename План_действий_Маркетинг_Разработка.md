# План действий — Маркетинг и Разработка

**Проект:** Pre-flight валидатор тендерной заявки
**Дата:** 2025-02-04

---

## Кратко

План действий для двух направлений: Маркетинг (как найти и убедить клиентов) и Разработка (как построить MVP v1.0).

---

## Направление 1: Маркетинг

### Цель
Найти 10-20 beta-пользователей за 30 дней, получить обратную связь, доказать ценность.

### Стратегия

**Каналы трафика:**

| Канал | Тип | Описание | Приоритет |
|---|---|---|---|
| **forum.gov-zakupki.ru** | Форум | Отвечать на вопросы, собирать живые формулировки отказов | P0 |
| **Telegram-каналы** | Группы | Присоединиться к тендерным чатам, давать советы | P0 |
| **LinkedIn** | B2B | Подходить к тендерным отделам компаний | P1 |
| **Email-аутрич** | Холодный | Письма тендерным отделам (с списком) | P2 |
| **Контент** | SEO/Статьи | Статьи на Хабр/VC/своем сайте | P1 |

**Офферы:**

| Оффер | Для кого | Формулировка |
|---|---|---|
| **Оффер "Страх ошибки"** | Тендерные отделы | "Сократите % отклонений по формалке с 15% до 2%. Проверьте заявку перед подачей — бесплатно." |
| **Оффер "Время"** | Тендерные отделы | "Экономия времени на подготовку заявки — 3 часа на каждую закупку." |
| **Оффер "Масштабирование"** | Тендерные агентства | "Автоматический pre-flight чек. 100% доведенных заявок. Возможность масштабирования без потери качества." |
| **Оффер "ROI"** | Собственники/КД | "Одна отклоненная заявка = потерянный контракт на 3,35 млн ₽. Pre-flight проверка стоит 20k ₽/мес." |

---

### Этап 1: Подготовка (неделя 1)

#### Задача 1.1: Создать landing page
**Что должно быть:**
- Заголовок: "Pre-flight валидатор для тендерщиков. Ловим 90% причин отклонений ДО подачи."
- Подзаголовок: "Не теряйте контракты из-за формальностей. Проверьте заявку перед подачей — бесплатно."
- Боли (3 шт): с описанием и решениями
- CTA: "Получить демо бесплатно"
- Форма: Email + Тип компании (отдел/агентство)

**Инструменты:**
- Webflow / Tilda / Framer
- Figma для дизайна

**Срок:** 5-7 дней

#### Задача 1.2: Написать офферы
**Для ICP-1 (тендерные отделы):**
- Email-шаблон (холодная рассылка)
- LinkedIn-сообщение
- Telegram-пост для групп

**Для ICP-2 (агентства):**
- Email-шаблон
- LinkedIn-сообщение

**Количество:** 3 оффера для каждого ICP

**Срок:** 2-3 дня

#### Задача 1.3: Создать профиль на форуме
**Действия:**
- Зарегистрироваться на forum.gov-zakupki.ru
- Заполнить профиль (эксперт по тендерам)
- Начать отвечать на вопросы (3-5 ответов/день)
- Цель: собрать живые формулировки отказов

**Срок:** 1 день

**Срок:** 3 недели

#### Задача 2.3: LinkedIn-аутрич

- Отправить 20-30 соединений/неделя

- Цель: 5-10 диалогов/неделя

**Метрика:** 5 диалогов, 2 демо/неделя

**Срок:** 3 недели

#### Задача 2.4: Email-рассылка (если будет база)
**Действия:**

**Метрика:** 10-15 открытий, 3-5 кликов, 1-2 демо

## Направление 2: Разработка

### Цель
Построить MVP v1.0 с P0 фичами за 8-12 недель.

### Стратегия

**Технологический стек (предложение):**

| Компонент | Технология | Почему |
|---|---|---|
| **Frontend** | React / Next.js | Популярно, есть специалисты |
| **Backend** | Node.js / Express | Совместимо с frontend |
| **База данных** | PostgreSQL | Надежно, есть JSONB для rule-engine |
| **Rule-engine** | JSON schema + custom validation | Гибко, легко расширять |
| **Интеграции** | Axios / node-fetch | HTTP-запросы к API |
| **Хостинг** | Vercel / Railway | Быстро, бесплатно для start |
| **Мониторинг** | Sentry / LogRocket | Отслеживание ошибок |

#### Задача 1.3: Разработать архитектуру
**Компоненты:**

```
Frontend (Next.js)
├── Pages
│   ├── Загрузка документов
│   ├── Проверка (progress)
│   └── Отчет (results)
├── Components
│   ├── FileUploader (drag & drop)
│   ├── ErrorPanel (список ошибок)
│   └── Checklist (чек-листы)
└── API calls

Backend (Node.js/Express)
├── Routes
│   ├── /api/upload (загрузка файлов)
│   ├── /api/validate (проверка)
│   └── /api/report (отчет)
├── Services
│   ├── AnonymityCheckService
│   ├── SignatureCheckService
│   ├── RequiredDocsService
│   └── Form2ValidationService
├── Integrations
│   ├── CryptoProAPI
│   ├── EGRULAPI
│   └── ETPIAPI
└── RuleEngine
    ├── Rules (JSON schema)
    └── Validator

Database (PostgreSQL)
├── Tables
│   ├── users (пользователи)
│   ├── checks (проверки)
│   └── errors (ошибки)
└── Views (аналитика)
```

**Срок:** 3-4 дня

## Риски и mitigations

### Маркетинг
| Риск | Mitigation |
|---|---|
| Низкий CTR офферов | A/B тест офферов, улучшать landing page |
| Нет базы email | Собирать на форуме и в Telegram |
| Не хотят демо | Сделать демо максимально простым (30 минут) |

### Разработка
| Риск | Mitigation |
|---|---|
| Интеграции сложно реализовать | Начать с mock API, позже реальный |
| Не успеть за 8 недель | Приоритизировать P0, P1 отложить |
| Нет команды разработки | Найти фрилансера / найм |
