## ТОП-3 проблемы (по Severity × Frequency)

### Проблема №1: Отклонения заявки из-за формальностей/несоответствий
**Severity:** 10/10 (проигрыш = 0 выручки)
**Frequency:** 7/10 (случается регулярно даже у опытных)
**Финансовый смысл:** средний контракт ~3,35 млн ₽

#### Симптомы
- Заявку отклонили "по формалке"
- Указали причину: "файл содержит информацию об участнике"
- Отклонили из-за "неполного комплекта документов"
- Заявку отклонили из-за "неверного формата файла"

#### Корневые причины
1. **Неанонимные документы** — в файле найден логотип, название компании, ИНН, телефон
2. **Неверный формат файла** — загружен файл, который не поддерживается (exe, rar и т.д.)
3. **Нет подписи** — документ не подписан КЭП
4. **Просроченный сертификат КЭП** — сертификат истек до подачи заявки
5. **Нет обязательного документа** — забыли прикрепить выписку из ЕГРЮЛ, доверенность и т.д.
6. **Лишний документ** — прикрепили то, что не требовалось по ТЗ
7. **Превышен размер файла** — файл больше лимита (обычно 50-100 МБ)
8. **Свежесть выписки из ЕГРЮЛ** — выписка старше 6 месяцев

#### Как мы решаем

| Решение | Фича | Приоритет |
|---|---|---|
| Автоматическое сканирование на анонимность (логотипы, названия, ИНН, телефоны) | 1.2.1 Проверка анонимности | P0 |
| Проверка форматов файлов | 1.1.3 Поддерживаемые форматы | P0 |
| Проверка размера файлов | 1.1.4 Лимит размера | P0 |
| Проверка наличия КЭП на всех документах | 1.3.1 Проверка наличия КЭП | P0 |
| Проверка валидности сертификата КЭП | 1.3.2 Валидность сертификата | P0 |
| Чек-лист обязательных документов по типу закупки | 2.2.1 Чек-лист документов | P0 |
| Проверка на лишние документы | 2.2.2 Проверка лишних документов | P1 |
| Проверка свежести выписки из ЕГРЮЛ | 2.2.4 Проверка свежести выписки из ЕГРЮЛ | P0 |
| Отчет по найденным ошибкам | 1.2.4 Отчет по анонимности | P0 |

#### Количество фич для решения: 8 фич (6 P0, 2 P1)

### Проблема №2: Форма 2 / заполнение характеристик / "не понимаю, что писать"
**Severity:** 9/10 (отклонение или дни потерь)
**Frequency:** 8/10 (почти в каждой закупке с ТЗ)
**Время на заполнение:** несколько дней

#### Симптомы
- Не понимаю, что писать в характеристике "???"
- Как правильно ответить: "указать значение..."
- Пустая трата времени.
- Что тут писать? (про типы/поля и требования)
- Помогите понять что писать в заявке(форме2)

#### Корневые причины
1. **Требования размазаны по документам** — данные разбросаны по разным файлам
2. **Не понятен тип характеристики** — числовая, текстовая, диапазон, булева?
3. **Не понятно правило** — "не хуже", "точнее", "или эквивалент" — что это значит?
4. **Единицы измерения** — в одном месте МПа, в другом кПа — как быть?
5. **Противоречия** — в одном месте 10 шт, в другом 11 шт — что правильно?
6. **Расплывчатые формулировки в ТЗ** — "от 5 до 10", "не хуже 1,6 МПа"
7. **Не указана страна происхождения** — обязательное поле, но забыли
8. **Не понятно, какие характеристики обязательные** — много полей, что выбрать?

#### Как мы решаем

| Решение | Фича | Приоритет |
|---|---|---|
| Извлечение требований из ТЗ (PDF/DOC) → структурированная таблица | 3.1.1 Парсинг PDF/DOC | P2 |
| Определение типа характеристики (числовая, текстовая, диапазон и т.д.) | 3.1.3 Определение типа характеристики | P2 |
| Определение правила ("не хуже", "точно", "или эквивалент") | 3.1.4 Определение правила | P2 |
| Проверка числовых значений (соответствие диапазону/точности) | 3.2.1 Проверка числовых значений | P1 |
| Проверка единиц измерения (соответствие ТЗ, конвертация) | 3.2.2 Проверка единиц измерения | P1 |
| Проверка текстовых значений (соответствие списку) | 3.2.3 Проверка текстовых значений | P1 |
| Проверка "не хуже" (значение ≤ или ≥) | 3.2.4 Проверка "не хуже" | P1 |
| Проверка обязательных полей | 3.2.5 Проверка обязательных полей | P0 |
| Проверка страны происхождения | 3.2.6 Проверка страны происхождения | P0 |
| Проверка на расплывчатые формулировки ("от и до", "более и менее") | 3.2.7 Проверка на расплывчатые формулировки | P0 |
| Проверка на противоречия в характеристиках | 3.2.8 Проверка на противоречия в характеристиках | P0 |
| Кросс-проверка документов и Формы 2 (поиск противоречий) | 3.3.1 Поиск противоречий | P1 |

### Проблема №3: Технические сбои ЕИС/ЭТП, КЭП, дедлайны
**Severity:** 8-9/10 (просрочка = ноль)
**Frequency:** 6/10 (регулярно у всех)

#### Симптомы
- Сроки подачи заявок не продлеваются... ситуация патовая
- Надо написать письмо... приложить скриншоты
- Не получается зайти... необходимо использовать квалифицированный сертификат
- Процедура на ЭТП зависла
- Надо чаще жалобы в Минфин отправлять

#### Корневые причины
1. **Просроченный сертификат КЭП** — сертификат истек, не заметили
2. **Нет доверенности** — подписывает сотрудник, а не директор, а доверенность нет
3. **Сбоя ЭТП/ЕИС** — площадка зависла в последний час
4. **Проблемы с входом** — сертификат не распознается, криптопровайдер не установлен
5. **Поздняя подача** — залили заявку за 5 минут до дедлайна, а площадка зависла
6. **Недостаточно средств** — не хватило средств для обеспечения заявки
7. **Не учли технический тайм-аут** — площадке нужно время на обработку (5-30 мин)

#### Как мы решаем

| Решение | Фича | Приоритет |
|---|---|---|
| Проверка валидности сертификата КЭП | 1.3.2 Валидность сертификата | P0 |
| Проверка полномочий подписанта (директор или по доверенности) | 1.3.3 Подписант vs Должность | P0 |
| Проверка доверенности (наличие, валидность) | 1.3.4 Проверка доверенности | P1 |
| Проверка свежести доверенности | 1.3.5 Проверка свежести доверенности | P1 |
| Проверка статуса аккаунта на ЭТП | 4.2.1 Статус аккаунта | P1 |
| Проверка баланса лицевого счета | 4.2.2 Баланс лицевого счета | P1 |
| Отслеживание дедлайна (время до конца подачи) | 4.3.1 Отслеживание дедлайна | P0 |
| Напоминания (24/6/1 час до дедлайна) | 4.3.2 Напоминания | P1 |
| Технический тайм-аут (предупреждение о времени обработки) | 4.3.3 Технический тайм-аут | P0 |
| "Режим подачи" с чек-листом готовности | 5.2.3 Чек-лист "Перед подачей" | P0 |

#### Количество фич для решения: 9 фич (4 P0, 5 P1)

## Дополнительные проблемы (вторичные)

### Проблема №4: Долгое время на сборку заявки
**Severity:** 6/10 (потерянное время)
**Frequency:** 8/10

#### Симптомы
- Сборка пакета заявки занимает часы/дни
- Не уверен, что всё включено
- Много версий документов — потерялся

#### Как мы решаем
- Автосборка "папки заявки" по шаблону (P1)
- Чек-лист обязательных документов (P0)
- Отчет по комплекту (P0)

---

### Проблема №5: Контроль качества внутри компании
**Severity:** 6/10 (риски ошибок)
**Frequency:** 6/10

#### Симптомы
- Коллеги проверяют... а то закалебали отклонять
- Ждем подписи/справки
- Версии файлов, хаос

#### Как мы решаем
- Сквозной процесс согласований (P2 — после PMF)
- Контроль версий (P2 — после PMF)
- Роли и права доступа (P2 — после PMF)

## Маркетинговое позиционирование

### Для сайта/лендинга

**Боли:**
- ❌ Заявку отклонили по формалке → потеряли контракт
- ❌ Дни потрачены на Форму 2 → не понял, что писать
- ❌ Просрочили дедлайн → сбой на площадке, подать не смогли

**Решения:**
- ✅ Pre-flight проверка — ловим 90% ошибок ДО подачи
- ✅ Чек-лист готовности — уверены, что всё ок
- ✅ Напоминания о дедлайнах — не забудете подать

**Заголовок:**
> "Pre-flight валидатор для тендерщиков. Ловим 90% причин отклонений ДО подачи."

**Подзаголовок:**
> "Не теряйте контракты из-за формальностей. Проверьте заявку перед подачей — бесплатно."
